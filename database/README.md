# Database

This folder is for Supabase database planning and future seed data.

## Purpose

Students can use this folder to think about tables before writing real SQL migrations.

## What To Fill In Later

1. Update `schema-plan.md` with the tables the first version truly needs.
2. Update `tables.md` with each table's columns in plain English.
3. Add real migration files only after the schema plan is clear.
4. Replace seed JSON placeholders with useful sample categories and riddles.

## Avoid

- Do not write SQL before the app flow is understood.
- Do not store real user data in seed files.
- Do not paste Supabase service role keys anywhere in this folder.

## Beginner Checklist

- [ ] Are planned tables easy to explain?
- [ ] Are sample seed files valid JSON?
- [ ] Are migrations still empty until the schema is ready?
